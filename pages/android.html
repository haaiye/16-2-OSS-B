안드로이드 아키텍처 개요

안드로이드 시스템은 소프트웨어 스택(stack)의 형태로 구성된다. 마치 블록을 쌓아놓은 것처럼 애플리케이션, 운영체제, 런타임 환경, 미들웨어, 각종 서비스와 라이브러리 등이 겹겹이 모여 구성된 것이다. 스택의 각 계층과 그것들의 요소들은 긴밀하게 통합 및 조율되어 있다.

1. 애플리케이션 : 안드로이드 소프트웨어 스택의 맨 위에 위치하고 있다. 특별한 안드로이드 애플리케이션으로 제공되는 기본 애플리케이션과 장치 구입 후 사용자가 설치한 애플리케이션으로 구성되는 것이 애플리케이션 계층이다.

2. 애플리케이션 프레임워크: 안드로이드 애플리케이션이 실행되고 관리되는 환경을 형성하는 서비스들이 모인 것이 애플리케이션 프레임워크이다. 안드로이드 애플리케이션들이 재사용과 상호 운용 및 교체 가능한 컴포넌트로 구성된다는 개념을 이 프레임워크가 구현한다. 액티비티 매니저, 콘텐츠 제공자, 리소스 매니저, 통지 매니저, 뷰 시스템, 패키지 매니저, 텔레포니 매니저, 위치 매니저라는 핵심 서비스를 제공한다.

3. 라이브러리 : 문자열 처리, 네트워킹, 파일 처리와 같은 일반적인 작업을 지원하기 위해 제공되는 표준 자바 라이브러리에 추가하여 안드로이드 개발 환경에는 안드로이드 라이브러리도 포함된다. 이것은 안드로이드 애플리케이션 개발에 특화된 다양한 자바 기반 라이브러리이다. 이 부류의 라이브러리 예로는 애플리케이션 프레임워크 라이브러리를 포함하여 사용자 인터페이스 생성, 그래픽 드로잉, 데이터베이스 액세스 등을 가능하게 해주는 라이브러리가 있다.

4. 안드로이드 런타임 : 안드로이드 스튜디오에서 안드로이드 애플리케이션이 빌드될 때에는 바이트 코드 형태(DEX,Dalvik Executable)로 컴파일 된다. 그리고 장치에 애플리케이션이 설치될 때 안드로이드 런타임(ART-Android RunTime)이 AOT(Ahead-0g-Time) 컴파일을 수행하여 바이트 코드를 장치의 프로세서(CPU)가 필요로 하는 네이티브 명령어로 모두 변환한다. 이렇게 변환된 형태를 ELF(Executable and Linkable Format)이라 한다. 따라서 애플리케이션이 실행될 때 ELF버전으로 실행되므로 애플리케이션의 실행 속도가 더 빠르고 배터리 수명도 항샹된다. 안드로이드 롤리팝 이전의 버전에서는 달빅 가상머신에서 바이트 코드를 하나씩 네이티브 명령어로 변환하면서 실행하였다.

5. 리눅스 커널 : 스택의 맨 아래에 위치한다. 리눅스 커널은 장치 하드웨어의 기반 운영체제 역할을 담당한다. 즉 멀티태스킹을 지원하고 메모리 관리와 프로세스 실행 및 관리 등을 처리하는 핵심 시스템 서비스를 비롯해서 네트워크 인터페이스와 각종 하드웨어 인터페이스를 위한 장치 드라이버(ex. 디스플레이, 와이파이, 오디오)를 제공한다.

안드로이드는 소프트웨어 스택 아키텍처의 형태로 구현되어 있다. 리눅스 커널, 런타임 환경과 그에 부합되는 라이브러리, 애플리케이션 프레임워크, 애플리케이션 등이 스택을 구성한다. 애플리케이션은 자바로 작성되고 안드로이드 스튜디오 빌드환경에서 바이트 코드 형태로 컴파일 된다. 그리고 나중에 애플리케이션이 장치에 설치될 때 안드로이드 런타임(ART)이 바이트코드를 장치의 프로세서(CPU)가 필요로 하는 네이티브 명령어로 일괄 변환한다.이 안드로이드 아키텍처의 목표는 성능과 효율이다.

안드로이드 스튜디오

안드로이드 애플리케이션은 기본적으로 Java로 개발하는데 그 개발도구로 안드로이드 스튜디오를사용한다. 안드로이드 애플리케이션 개발 뿐만이 아니라 프로그램을 만들 때 ‘통합개발환경(IDE: Integrated Devlopment Environment)’라는 도구를 사용하는 것이 일반적이다. IDE에는 몇 가지 종류가 있지만 몇 년 전까지 주로 사용한 것은 이클립스(Eclipse)이다, 최근에는 구글이 개발한 IDE인 ‘인텔리제이 아이디어(IntelliJ IDEA)’를 주로 사용하고 있다.

개발환경 구축하기

1. 개발용 컴퓨터 : 윈도우, 리눅스, OS X(Mac)등의 OS에서 작업이 가능하다.

2. JDK(Java Development Kit) : JDK는 오라클에서 발매한 무료 소프트웨어로, Java의 컴파일 환경, 실행 환경(JRE), 디버그 등이 들어있다.

3. 안드로이드 스튜디오 : Java나 XML 에디터, 컴파일, 디버그, 공개용 파일 작성 등 안드로이드 애플리케이션 개발에 필요한 도구가 모두 들어있다.

4. 안드로이드 SDK(Software Development Kit) : 안드로이드 SDK는 안드로이드 애플리케이션 개발에 필수인 라이브러리와 각종 도구의 세트를 말한다. 안드로이드 버전 별 라이브러리, 단말기를 USB에 연결하여 개발하는 데 필요한 것, 단말기가 없는 경우를 대비한 안드로이드 단말기 에뮬레이터 등이 들어있다. 이들은 안드로이드 스튜디오 내려받기 시 함께 포함되어 있다.

5. 안드로이드 단말기 : GPS나 지자기 센서 등의 하드웨어를 이용하는 애플리케이션을 개발하려고 한다면 실제 기기가 필수이다. 개발용 에뮬레이터를 사용해도 되지만 동작이 매우 느리다는 단점이 있다.

6. ADB(Android Debug Bridge) : ADB는 개발에 사용하는 컴퓨터를 ADB서버로, 안드로이드 단말기를 ADB클라이언트로 동작시키는 개발 도구이다. 이 도구를 이용하면 ADB 서버와 ADB클라이언트 사이에서 통신이 가능하다.

프로젝트

프로젝트는 안드로이드 스튜디오로 애플리케이션을 개발할 때 가장 먼저 만드는 것이다. 이 곳에 리소스나 Java 프로그램을 저장한다. 일반적인 시스템 개발에서는 여러 개의 하위 시스템을 만들고 그것을 모두 연계하여 하나의 시스템을 구축하는데 안드로이드 스튜디오는 이러한 형태로 애플리케이션을 개발할 수 있도록 설계되어 있으므로 하나의 프로젝트 안에 여러 개의 애플리케이션을 만들 수 있다. 이렇게 여러 개의 애플리케이션이나 프로그램을 묶는 것은 프로젝트의 기능 중 하나이다.
